# Milestone v2.2: Team Updates, Knowledge Sharing & Weekly Board

**Status:** ✅ SHIPPED 2026-04-28
**Phases:** 23-25
**Total Plans:** 12

## Overview

v2.2 delivered the team communication layer: structured standup updates with frozen task snapshots, a knowledge-sharing scheduler inside the existing calendar flow, and a weekly markdown board with AI-generated summaries.

## Phases

### Phase 23: Standup Updates

**Goal**: Members can post structured daily/weekly standups with a frozen task snapshot, and the whole team can browse and filter the feed
**Depends on**: Phase 22 (v2.1 structure baseline)
**Plans**: 5 plans

Plans:

- [x] 23-01: SQLAlchemy models, Pydantic schemas, Alembic migration
- [x] 23-02: Frontend packages, updates API module, updates store
- [x] 23-03: Backend router: all 6 endpoints + main.py registration
- [x] 23-04: Frontend UI: page, form, card, snapshot panel, nav item
- [x] 23-05: Standup template field types + rich text support

**Details:**
- Added `StandupPost`, `StandupTemplate`, and `StandupSettings` as the backing data model.
- Built `/api/updates` endpoints for template management, feed browsing, create/update/delete, and server-side snapshot capture.
- Delivered the `/updates` page, standup form, post card, snapshot panel, and navigation entry.
- Extended standup templates with field type support for richer inputs and display.

### Phase 24: Knowledge Sharing Scheduler

**Goal**: Admins and supervisors can schedule Knowledge Sharing sessions scoped to their team, and members receive notifications for sessions in their scope
**Depends on**: Phase 23
**Plans**: 3 plans

Plans:

- [x] 24-01: Backend foundation: KnowledgeSession enums, model, schemas, Alembic migration
- [x] 24-02: Backend behavior: scoped router, presenter validation, notification fanout, reminder sync, backend tests
- [x] 24-03: Frontend `/schedule` integration: Knowledge Sessions tab, agenda/calendar views, scoped modal UX, frontend verification

**Details:**
- Added a dedicated `KnowledgeSession` domain separate from personal schedules.
- Added scoped create/list/update/delete behavior and notification/reminder fanout.
- Integrated Knowledge Sessions into the existing `/schedule` page as a dedicated tab without a new route.

### Phase 25: Team Weekly Board & AI Summary

**Goal**: Any member can post markdown updates to a shared weekly board, and an AI summary is available both on demand and automatically at end of week
**Depends on**: Phase 24
**Plans**: 4 plans

Plans:

- [x] 25-01: Backend foundation: weekly post, append-entry, and summary models/schemas plus Alembic migration
- [x] 25-02: Backend board behavior: week payload, primary post CRUD, append-entry CRUD, ownership tests
- [x] 25-03: AI summary behavior: shared summary service, cooldown/cache logic, Sunday 23:00 scheduler job, summary tests
- [x] 25-04: Frontend `/board` experience: nav entry, API/store, week navigator, summary panel, markdown composer, digest cards, frontend verification

**Details:**
- Added a weekly board with one primary post per member per ISO week and append entries for follow-ups.
- Added a cached weekly AI summary flow with cooldown protection and Sunday 23:00 automation.
- Delivered the `/board` frontend experience with sanitized markdown rendering and author-only mutations.

## Milestone Summary

**Decimal Phases / Inserted Work**

- Phase 23.05: Standup template field types and richer display support were added as follow-on work.

**Key Decisions**

- Standup snapshots are server-side JSONB so historical posts remain immutable.
- Knowledge Sessions use a dedicated `KnowledgeSession` table and separate router rather than mutating personal schedules.
- `/schedule` gains a tab-based split for personal events vs. knowledge sessions, preserving the existing route.
- Weekly board summaries are cached per ISO week with a 30-minute manual cooldown.
- Markdown rendering uses `marked` + `DOMPurify` before any `{@html}` usage.

**Issues Resolved**

- Fixed async auth-cookie interference in test flows by moving verification to direct bearer-token generation.
- Verified the Knowledge Sessions scope and reminder flows with focused backend tests.
- Kept the existing scheduler and notification pipeline authoritative rather than introducing a parallel reminder system.

**Issues Deferred**

- Manual browser UAT remains a recommended confidence step after the automated checks.
- Future planning work is moving into the next milestone rather than expanding v2.2 further.

**Technical Debt Incurred**

- Some broader app warnings remain outside this milestone in unrelated frontend routes.
- The project still relies on manual browser verification for certain interaction-heavy flows.
