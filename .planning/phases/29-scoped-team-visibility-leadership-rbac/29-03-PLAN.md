<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser to list endpoints | Users can request broad collections that may contain hidden teams or users |
| browser to direct-ID endpoints | Users can guess IDs for hidden projects, tasks, milestones, posts, sessions, or notifications |
| reporting endpoints to aggregate data | Performance/timeline/board summaries may leak hidden users through counts or nested payloads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-07 | I | project/task/timeline/milestone routers | mitigate | Apply shared visible-project/task filters to both list and direct-ID endpoints, then test blocked cross-scope access |
| T-29-08 | I | updates/board/knowledge/performance payloads | mitigate | Replace local old-role filters with shared visibility helpers and test aggregate payloads for hidden-row leaks |
| T-29-09 | E | performance and management-like surface access | mitigate | Update privileged access checks to manager/leader helpers and remove old admin dependency paths |
</threat_model>
