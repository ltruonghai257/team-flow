- normalize `start_time` with the same `_to_naive_utc` pattern used in `work.py`
- strip each tag, remove blanks, and deduplicate case-insensitively while preserving first-entry display casing
- require `duration_minutes >= 15`
- accept empty `offset_minutes_list`

Represent `tags` in API responses as `list[str]`, not a comma string, so the frontend chip input can bind directly to the response shape.
