  <action>
Create `backend/app/routers/knowledge_sessions.py` and register it in `backend/app/api/main.py`.

Implement these exact endpoints:

- `GET /api/knowledge-sessions/`
  - optional query params: `start`, `end`
  - returns sessions ordered by `start_time`
  - members: org-wide rows plus rows where `sub_team_id == current_user.sub_team_id`
  - supervisors: org-wide rows plus rows where `sub_team_id` is the supervisor's managed sub-team
  - admins with `X-SubTeam-ID`: org-wide rows plus that sub-team's rows
  - admins without `X-SubTeam-ID`: all sessions

- `POST /api/knowledge-sessions/`
  - requires `require_supervisor_or_admin`
  - if current user is supervisor, force `sub_team_id` to the supervisor's sub-team ID
  - if current user is admin and no `X-SubTeam-ID` header is active, create org-wide session with `sub_team_id = None`
  - if `presenter_id` is missing, set it to `current_user.id`
  - validate presenter belongs to allowed scope
  - normalize and store tags via `serialize_tags`
  - create notification rows through `sync_knowledge_session_notifications`

- `GET /api/knowledge-sessions/{session_id}`
  - return 404 when the session exists but is outside the requester's scope

- `PATCH /api/knowledge-sessions/{session_id}`
  - requires `require_supervisor_or_admin`
  - enforce scope ownership rules from create
  - if `start_time` or `offset_minutes_list` changes, resync pending reminders
  - do not emit a second creation broadcast

- `DELETE /api/knowledge-sessions/{session_id}`
  - requires `require_supervisor_or_admin`
  - delete the session
  - remove pending knowledge-session notification rows for that event before commit

Use `KnowledgeSessionOut` responses that deserialize the stored comma string back to `list[str]` for `tags`.
  </action>
  <acceptance_criteria>
    - `backend/app/routers/knowledge_sessions.py` contains `prefix="/api/knowledge-sessions"`
    - `backend/app/routers/knowledge_sessions.py` contains `@router.get("/")`
    - `backend/app/routers/knowledge_sessions.py` contains `@router.post("/")`

    - `backend/app/api/main.py` contains `application.include_router(knowledge_sessions.router)`
