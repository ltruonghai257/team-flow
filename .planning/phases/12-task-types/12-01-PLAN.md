<threat_model>
## Threat Model

| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-12-01 | Invalid task type values are persisted through API or AI input | medium | Use backend enum validation/coercion; only accept `feature`, `bug`, `task`, `improvement`; default AI invalid/missing values to `task`. |
| T-12-02 | Existing task rows get null type after migration | high | Add Alembic migration with non-null column and server default `task`; verify migration contains `nullable=False` and `server_default`. |
| T-12-03 | Type filter leaks or breaks task list by accepting arbitrary SQL-like strings | medium | Parse type filters into enum values before applying `Task.type.in_(...)`; reject invalid values with `HTTPException(status_code=422)`. |
</threat_model>

<context>

- `feature`

Display labels:
- `feature` -> `Feature`

- `improvement` -> `Improve`

API filter contract:
- Use query param `types=feature,bug`
- Empty or missing `types` means all types
- Invalid type tokens return `422` with a clear detail message
