Add endpoint:
`@router.get("/kpi/overview", response_model=KPIOverviewResponse)`

It must return:
- `scorecards`: member KPI scorecards for users in the current sub-team scope
- `needs_attention`: subset of scorecards with score below 70 or warning/critical reason labels
- `summary`: totals for active tasks, completed tasks, average score, average cycle time, and defect count
- `weights`: current KPI weights

Use `_get_or_create_kpi_weights`.

Default scoring:
- Workload balance: 100 when active tasks <= 7, 70 when 8-10, 40 when > 10
- Completion/velocity: min(100, completed_30d * 10)
- Cycle time: 100 when avg_cycle_time_hours <= 48, 70 when <= 120, 40 when > 120 or missing
- On-time rate: rounded on-time percentage
- Defect/quality: 100 when bug MTTR <= 72 hours or no bugs, 70 when <= 168, 40 when > 168

Weighted KPI score = sum(category_score * weight / 100), rounded to integer.
