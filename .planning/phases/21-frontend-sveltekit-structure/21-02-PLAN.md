<task id="21-02-01" type="execute">
  <name>Create `apis/request.ts` with protected request behavior</name>
  <files>frontend/src/lib/apis/request.ts</files>
  <read_first>frontend/src/lib/api.ts, frontend/src/lib/stores/subTeam.ts, .planning/phases/21-frontend-sveltekit-structure/21-PATTERNS.md</read_first>
  <action>
Create `frontend/src/lib/apis/request.ts`.

Move the request wrapper implementation from `frontend/src/lib/api.ts` into this file with the same observable behavior:

- `const BASE = '/api';`
- import `subTeamStore` from `$lib/stores/subTeam`
- define `interface SubTeam` with `id`, `name`, and `supervisor_id`
- export `interface ApiError extends Error` with optional `detail`, `status`, and `payload`
- export `async function request<T = any>(path: string, options: RequestInit = {}): Promise<T>`
- always set `Content-Type: application/json`
- read the selected sub-team by subscribing once to `subTeamStore` and then unsubscribing
- when selected sub-team has an id, add `headers['X-SubTeam-ID'] = activeSubTeam.id.toString()`
- call `fetch(`${BASE}${path}`, { ...options, headers, credentials: 'include' })`
- on non-OK responses, parse JSON fallback `{ detail: res.statusText }`, build `Error(message) as ApiError`, and set `detail`, `status`, and `payload`
- return `undefined as T` for HTTP 204
- return `res.json()` otherwise
  </action>
  <acceptance_criteria>
    - `frontend/src/lib/apis/request.ts` exists
    - It contains `const BASE = '/api'`
    - It imports `subTeamStore`
    - It contains `credentials: 'include'`
    - It contains `X-SubTeam-ID`
    - It contains `res.status === 204`
    - It exports `request`
  </acceptance_criteria>
  <verify>`rtk rg "BASE = '/api'|subTeamStore|credentials: 'include'|X-SubTeam-ID|res.status === 204|export async function request" frontend/src/lib/apis/request.ts`</verify>
  <done>Shared request behavior has a canonical module under `src/lib/apis`.</done>
</task>

<task id="21-02-02" type="execute">
  <name>Import request wrapper from api.ts</name>
  <files>frontend/src/lib/api.ts</files>
  <read_first>frontend/src/lib/api.ts, frontend/src/lib/apis/request.ts</read_first>
  <action>
Update `frontend/src/lib/api.ts` to import the shared wrapper from the new canonical module:

`import { request } from '$lib/apis/request';`

Remove the local definitions that moved to `apis/request.ts`:

- `const BASE = '/api';`
- local `interface SubTeam`
- local `interface ApiError`
- local `async function request<T = any>(...)`

Keep `auth.login` behavior intact. Since `auth.login` still uses raw `fetch`, preserve its exact target and options:

- `fetch('/api/auth/token', { method: 'POST', body, headers: { 'Content-Type': 'application/x-www-form-urlencoded' }, credentials: 'include' })`

If the implementation needs `BASE` only for `auth.login`, either import a public constant from `request.ts` or replace the target with the literal `'/api/auth/token'`. Do not duplicate general request-wrapper logic in `api.ts`.
  </action>
  <acceptance_criteria>
    - `api.ts` imports `request` from `$lib/apis/request`
    - `api.ts` no longer defines `async function request`
    - `api.ts` still exports all existing API namespaces
    - `auth.login` still posts to `/api/auth/token`
    - No route/component/store imports are changed in this plan
  </acceptance_criteria>
  <verify>`rtk rg "\\$lib/apis/request|/api/auth/token|export const auth|export const tasks|export const statusSets" frontend/src/lib/api.ts && rtk rg "async function request|interface ApiError|interface SubTeam" frontend/src/lib/api.ts` should return no matches for the second command; then run `cd frontend && bun run check` if Bun is available</verify>
