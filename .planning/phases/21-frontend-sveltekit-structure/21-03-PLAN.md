- `status-sets.ts` exporting `statusSets`

- `POST /api/auth/token`
- `Content-Type: application/x-www-form-urlencoded`
- `credentials: 'include'`
- throws `new Error((await r.json()).detail || 'Login failed')` when response is not OK

Other auth methods use `request`:

- `/auth/register`
- `/auth/me`
- `/auth/logout`

- `/status-sets/default`
- `/status-sets/effective`
- `/status-sets/${statusSetId}/transitions`

- `/status-sets/statuses/${statusId}`
- `/status-sets/${statusSetId}/reorder`
- `/status-sets/statuses/${statusId}/delete`
- `/status-sets/projects/${projectId}/override`

Preserve sub-team/reminder endpoint paths:

- `/sub-teams/`
- `/sub-teams/${id}`
- `/sub-teams/reminder-settings/current`
- `/sub-teams/reminder-settings/proposals`
- `/sub-teams/reminder-settings/proposals/${id}/review`

Use `import type` from `$lib/types/status` and `$lib/types/notification` where needed.
  </action>
  <acceptance_criteria>
