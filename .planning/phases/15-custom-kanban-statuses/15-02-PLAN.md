# Plan 15-02 — Backend Status APIs and Task Dual-Write

<objective>
Expose status-set management APIs and update task create/update/list behavior so DB-backed statuses drive completion logic while legacy enum input remains compatible.
</objective>

<must_haves>
- D-01, D-02, D-03: default status management is sub-team scoped and admin switcher aware.
- D-04, D-05, D-06: project overrides inherit defaults, use stable slugs, and revert by slug with fallback for unmatched statuses.
- D-07, D-08, D-09: `is_done` drives task completion and prepares Phase 16.
- D-14, D-15, D-16: assigned statuses are moved or archived safely and reorder saves immediately.
- D-17, D-18: AI and legacy status input continue to work via backend mapping.
</must_haves>

<task id="15-02-03" type="execute">
  <name>Implement safe delete, archive, reorder, and project override behavior</name>
  <read_first>backend/app/routers/statuses.py, .planning/phases/15-custom-kanban-statuses/15-CONTEXT.md, .planning/phases/15-custom-kanban-statuses/15-RESEARCH.md</read_first>
  <action>
In `backend/app/routers/statuses.py`, enforce:

- Reorder accepts the complete ordered list for one status set and rewrites `position` from `0` upward.
- Updating `name` must not silently change `slug` for existing statuses.
- `mode = "archive"` sets `is_archived = True` and leaves assigned tasks unchanged.
- `mode = "move_delete"` requires `replacement_status_id`, verifies replacement is in the same status set and not archived, updates tasks from old status to replacement, then deletes old status.
- If `task_count > 0` and `mode = "delete"` without replacement, return 400 with `Statuses with tasks must be moved or archived before they can be deleted.`
- Project override creation copies effective statuses into a project-scoped `StatusSet`.
- Project override revert auto-maps by matching slug. If any project status slug has no default match and no `fallback_mappings[project_status_id]`, return 400 with details for unmatched statuses. When mappings are complete, move tasks to default statuses and delete the project override set.
  </action>
  <acceptance_criteria>
    - `backend/app/routers/statuses.py` contains `mode == "archive"`
    - `backend/app/routers/statuses.py` contains `mode == "move_delete"`
    - `backend/app/routers/statuses.py` contains `replacement_status_id`
    - `backend/app/routers/statuses.py` contains `Statuses with tasks must be moved or archived before they can be deleted.`
    - `backend/app/routers/statuses.py` contains `fallback_mappings`
    - `backend/app/routers/statuses.py` contains `slug`
  </acceptance_criteria>
</task>
