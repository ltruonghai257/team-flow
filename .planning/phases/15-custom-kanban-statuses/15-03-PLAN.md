# Plan 15-03 — Frontend API and Reusable Status Components

<objective>
Add the frontend API client and reusable status-management components that implement the approved UI-SPEC without wiring them into pages yet.
</objective>

<must_haves>
- D-10, D-11, D-12: status management is reachable from both tasks and projects.
- D-13: mixed project board states must not expose project-specific settings.
- D-14, D-15, D-16: safe delete/archive and immediate reorder are supported.
- UI-SPEC: no new UI packages; use existing Tailwind primitives and `lucide-svelte`.
</must_haves>

<tasks>

<task id="15-03-01" type="execute">
  <name>Add statusSets API client methods</name>
  <read_first>frontend/src/lib/api.ts, .planning/phases/15-custom-kanban-statuses/15-RESEARCH.md</read_first>
  <action>
In `frontend/src/lib/api.ts`, add exported TypeScript types:

- `CustomStatus`
- `StatusSet`
- `StatusSetScope`

Add `export const statusSets = { ... }` with methods:

- `defaultSet: () => request('/status-sets/default')`
- `effective: (projectId?: number) => request('/status-sets/effective' + query)`
- `createDefaultStatus: (data) => request('/status-sets/default/statuses', { method: 'POST', body: JSON.stringify(data) })`
- `updateStatus: (id, data) => request('/status-sets/statuses/${id}', { method: 'PATCH', body: JSON.stringify(data) })`
- `reorder: (statusSetId, statusIds) => request('/status-sets/${statusSetId}/reorder', { method: 'POST', body: JSON.stringify({ status_ids: statusIds }) })`
- `deleteOrArchiveStatus: (id, data) => request('/status-sets/statuses/${id}/delete', { method: 'POST', body: JSON.stringify(data) })`
- `createProjectOverride: (projectId) => request('/status-sets/projects/${projectId}/override', { method: 'POST' })`
- `revertProjectOverride: (projectId, fallbackMappings) => request('/status-sets/projects/${projectId}/override', { method: 'DELETE', body: JSON.stringify({ fallback_mappings: fallbackMappings }) })`

Use the existing `request()` helper so sub-team headers continue to work.
  </action>
  <acceptance_criteria>
    - `frontend/src/lib/api.ts` contains `export interface CustomStatus`
    - `frontend/src/lib/api.ts` contains `export interface StatusSet`
    - `frontend/src/lib/api.ts` contains `export const statusSets`
    - `frontend/src/lib/api.ts` contains `/status-sets/effective`
    - `frontend/src/lib/api.ts` contains `fallback_mappings`
  </acceptance_criteria>
</task>
