# Plan 15-04 — Integrate DB Statuses into Tasks, Kanban, and Projects UI

<objective>
Make `/tasks`, the Kanban board, and `/projects` consume DB-backed status records and expose the approved management surfaces.
</objective>

<must_haves>
- D-10: status management available from `/projects` and `/tasks`.
- D-11: `/projects` provides full project-context override settings.
- D-12: `/tasks` provides direct Kanban column management.
- D-13: mixed project views do not expose project-specific settings.
- UI-SPEC: Kanban columns render active status records by `position`, preserve mobile horizontal scroll, and use status colors.
</must_haves>

<tasks>

<task id="15-04-01" type="execute">
  <name>Update KanbanBoard to render DB-backed statuses</name>
  <read_first>frontend/src/lib/components/tasks/KanbanBoard.svelte, frontend/src/lib/components/tasks/KanbanCard.svelte, frontend/src/lib/utils.ts, .planning/phases/15-custom-kanban-statuses/15-UI-SPEC.md</read_first>
  <action>
Update `KanbanBoard.svelte`:

- Add exported prop `statuses: CustomStatus[] = []`.
- Remove `const columns = ['todo', 'in_progress', 'review', 'done', 'blocked'];` as the primary source of truth.
- Build columns from active statuses: `statuses.filter((s) => !s.is_archived).sort((a, b) => a.position - b.position)`.
- Group tasks by `task.custom_status_id` when present.
- Fallback to legacy `task.status` only when `custom_status_id` is missing.
- On drag finalize, call `onStatusChange(moved.id, targetStatus.id)` where target is DB status ID.
- Header renders color dot, name, count, and `Done` pill when `is_done`.
- Preserve wrapper style `touch-action: pan-x pan-y;`.
- Empty column copy is `No tasks in this status`.
  </action>
  <acceptance_criteria>
    - `frontend/src/lib/components/tasks/KanbanBoard.svelte` contains `export let statuses`
    - `KanbanBoard.svelte` does not contain `const columns = ['todo', 'in_progress', 'review', 'done', 'blocked']`
    - `KanbanBoard.svelte` contains `custom_status_id`
    - `KanbanBoard.svelte` contains `is_done`
    - `KanbanBoard.svelte` contains `No tasks in this status`
    - `KanbanBoard.svelte` contains `touch-action: pan-x pan-y`
  </acceptance_criteria>
</task>
