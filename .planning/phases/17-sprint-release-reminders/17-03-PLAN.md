  <name>Make rebuild helpers preserve historical notifications</name>

<task id="17-03-02" type="execute">
  <name>Rebuild sprint reminders on sprint create/update</name>
  <read_first>backend/app/routers/sprints.py, backend/app/services/reminder_notifications.py</read_first>
  <action>
Update `backend/app/routers/sprints.py`:

- After creating a sprint with `end_date`, call `rebuild_sprint_reminders(db, sprint.id)` before refresh/return.
- On update, detect whether `end_date` changed.
- If `end_date` changed, call `rebuild_sprint_reminders(db, sprint.id)` after applying payload changes and before returning.
- Do not rebuild on task assignment changes; scheduled reconciliation handles membership drift.
  </action>
  <acceptance_criteria>
    - `backend/app/routers/sprints.py` imports `rebuild_sprint_reminders`
    - Create path calls `rebuild_sprint_reminders` when `end_date` exists
    - Update path detects `end_date` in payload and rebuilds
    - No task assignment hook is added
  </acceptance_criteria>
</task>
