<task id="17-02-01" type="execute">
  <name>Create reminder generation service</name>
  <read_first>backend/app/models.py, backend/app/scheduler_jobs.py, backend/app/routers/notifications.py</read_first>
  <action>
Create `backend/app/services/reminder_notifications.py`.

Implement helpers:

- `async def get_or_create_reminder_settings(session, sub_team_id: int) -> SubTeamReminderSettings`
- `async def rebuild_sprint_reminders(session, sprint_id: int) -> int`
- `async def rebuild_milestone_reminders(session, milestone_id: int) -> int`
- `async def reconcile_generated_reminders(session) -> int`

Service behavior:

- Skip sprint reminders when `Sprint.end_date` is null.
- Derive sprint participants from tasks with `Task.sprint_id == sprint.id` and non-null `assignee_id`.
- Derive milestone/project participants from tasks in the milestone/project with non-null `assignee_id`.
- Add immediate `SubTeam.supervisor_id` for every participant's sub-team when present.
- Dedupe recipients by user id.
- If a user qualifies as both participant and supervisor, create one notification with participant wording.
- Respect sub-team settings for lead time and sprint/milestone toggles.
- Calculate `remind_at = event_date - timedelta(days=lead_time_days)`.
- If `remind_at` is in the past, store `remind_at = now`.
- Insert pending `EventNotification` rows and rely on the Phase 17 partial unique index for duplicate protection.
  </action>

    - File contains `reconcile_generated_reminders`
    - File queries `Task.assignee_id`
    - File references `SubTeam.supervisor_id`
  </acceptance_criteria>
</task>
