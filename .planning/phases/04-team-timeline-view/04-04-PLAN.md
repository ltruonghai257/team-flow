<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete Team Timeline View:
- Backend: GET /api/timeline (projects + milestones + tasks + unassigned_tasks)
- Frontend: /timeline route with Gantt chart, toolbar, view toggle, range selector, drag-to-reschedule, click-to-edit modal
  </what-built>
  <how-to-verify>
**Setup:** Ensure backend and frontend dev servers are running.

**1. Page loads**
- Navigate to `http://localhost:5173/timeline`
- Expected: Page renders with "Team Timeline" header, toolbar visible, no 404 or white screen

**2. Gantt chart renders (D-01)**
- Expected: Horizontal bars visible for tasks, grouped by project → milestone → task rows (project view)
- If no data: Create a project with milestones and tasks via the Projects/Tasks pages first

**3. Color coding (D-05)**
- Expected: Task bars use the project color (visible as bar background or border color)

**4. View toggle (D-03)**
- Click "By Member" button in toolbar
- Expected: Rows reorganize to show one row per team member, tasks shown across projects

**5. Unscheduled task dashed bars (D-10)**
- Create or identify a task with no due_date
- Expected: That task appears as a dashed/outlined bar at today's date position — NOT hidden

**6. Time range selector (D-09)**
- Select "Week" from range dropdown → Gantt zooms to current week
- Select "Month" → Gantt shows current month
- Select "Custom" → Two date inputs appear; change dates → Gantt window updates

**7. Drag to reschedule (D-02)**
- Drag a task bar to a new date
- Expected: Bar moves, toast notification appears ("Rescheduled to [date]"), backend PATCH fires (check network tab — PATCH /api/tasks/{id} with due_date)

**8. Click to edit modal (D-06)**
- Click a task bar
- Expected: Modal opens with task title, status, priority, due_date, description fields
- Edit the title, click Save
- Expected: Toast "Task updated", modal closes, Gantt refreshes with new data

**9. Overdue task indicator (Claude's Discretion)**
- Expected: Tasks past their due_date and not done show a red outline

**10. Sidebar navigation**
- Expected: "Timeline" with GanttChartSquare icon appears in left sidebar
- Expected: Clicking it from any page navigates to /timeline and highlights the nav item
  </how-to-verify>
  <resume-signal>Type "approved" if all checks pass, or describe any issues found (e.g., "drag not working", "modal not opening")</resume-signal>
