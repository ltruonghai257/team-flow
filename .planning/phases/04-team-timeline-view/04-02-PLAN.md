
yarn 1.x does NOT fail on peer dependency conflicts — the `svelte@^4.0.0` peer declaration in svelte-gantt will produce a warning but the install will succeed.

After install, verify the package is importable:
```bash
cd frontend && node -e "require('./node_modules/svelte-gantt/index.js'); console.log('svelte-gantt OK')"
```

If this import test fails with a runtime module error (e.g., "Cannot find module 'svelte/internal'"), this indicates Svelte 5 runtime incompatibility. In that case:
1. Remove svelte-gantt: `yarn remove svelte-gantt`
2. Install the Svelte 5 native fallback: `yarn add @svar-ui/svelte-gantt`
3. Note this decision in the SUMMARY.md — D-01 specified svelte-gantt but Svelte 5 incompatibility required the fallback
4. The Gantt wrapper component in Plan 03 will need to use `@svar-ui/svelte-gantt` import syntax instead

The PLAN.md for Plan 03 accounts for both import paths.
