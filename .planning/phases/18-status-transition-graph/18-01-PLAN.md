<task id="18-01-03" type="execute">
  <name>Add transition list and replace endpoints</name>
  <read_first>backend/app/routers/statuses.py, backend/app/schemas.py, backend/app/models.py</read_first>
  <action>
Extend `backend/app/routers/statuses.py` under the existing `/api/status-sets` router:

- `GET /{status_set_id}/transitions?include_archived=false`
  - Return transitions for the given status set.
  - Default response includes only transitions whose `from` and `to` statuses are active.
  - `include_archived=true` returns archived-linked transitions for management/recovery.
- `POST /{status_set_id}/transitions`
  - Supervisor/admin only through `_require_status_write_scope`.
  - Replace the full transition set atomically from `StatusTransitionsReplace`.
  - Reject endpoints that do not both belong to `status_set_id`.
  - Reject archived endpoint statuses on writes.
  - Reject self-transitions.
  - Preserve empty list as a valid saved state.
- `DELETE /{status_set_id}/transitions/{transition_id}`
  - Supervisor/admin only through `_require_status_write_scope`.
  - Delete only a transition belonging to `status_set_id`.
  - Return 404 for missing or cross-set transition IDs.

Keep this in the existing status-set router unless file size forces a small helper module; avoid new router registration unless necessary.
  </action>
  <acceptance_criteria>
    - `backend/app/routers/statuses.py` contains `/{status_set_id}/transitions`
    - GET accepts `include_archived`
    - POST calls `_require_status_write_scope`
    - DELETE removes one transition by ID under the target status set
    - POST validates both endpoints are in the same status set
    - POST allows an empty transition list
  </acceptance_criteria>
</task>
