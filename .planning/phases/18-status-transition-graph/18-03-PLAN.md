<task id="18-03-03" type="execute">
  <name>Add read-only graph preview and warnings</name>
  <read_first>.planning/phases/18-status-transition-graph/18-UI-SPEC.md, frontend/src/lib/components/statuses/StatusTransitionEditor.svelte</read_first>
  <action>
Create `frontend/src/lib/components/statuses/StatusTransitionPreview.svelte` or an internal child if keeping it in one file is simpler.

Preview requirements:

- Show active statuses as nodes using status color dots.
- Show allowed transitions as simple edges or compact `from -> to` links.
- Read-only only; no drag-to-connect or graph editing.
- Empty preview copy: `Save or select transitions to preview the workflow.`
- Handle long status names with truncation and title text.

Warnings:

- Non-blocking warning list headed `Workflow warnings`.
- Warn when transition rules exist and active statuses have no outgoing transitions.
- Warn when no `is_done` status is reachable if feasible with small helper logic.
- Do not block save for warnings.
  </action>
  <acceptance_criteria>
    - Preview uses status names and colors
    - Preview does not expose editing controls
    - Warnings are non-blocking
    - Long names cannot overflow the preview container
  </acceptance_criteria>
</task>
