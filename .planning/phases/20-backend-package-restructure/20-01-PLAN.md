<task id="20-01-04" type="execute">
  <name>Introduce canonical app factory target and main compatibility delegate</name>
  <files>backend/app/api/main.py, backend/app/main.py</files>
  <read_first>backend/app/main.py, backend/app/routers, backend/app/core/config.py, backend/app/internal/scheduler_jobs.py</read_first>
  <action>
Move app assembly into `backend/app/api/main.py`.

`app.api.main` must expose:

- `run_migrations()`
- `create_app()`
- module-level `app = create_app()`

`create_app()` must preserve existing behavior:

- FastAPI title/description/version
- lifespan behavior including optional Alembic upgrade, scheduler startup, and scheduler shutdown
- rate limiter state and exception handler
- CORS middleware settings
- all current router registrations in the same effective order
- `/health` returning `{"status": "ok"}`

Replace `backend/app/main.py` with a compatibility delegate that imports and re-exports `app`, `create_app`, and `run_migrations` from `app.api.main`.
  </action>
  <acceptance_criteria>
    - `backend/app/api/main.py` contains `def create_app(`
    - `backend/app/api/main.py` exports module-level `app`
    - `backend/app/main.py` re-exports `app`, `create_app`, and `run_migrations`
    - `/health` behavior is preserved
    - Router registration still includes every current router
  </acceptance_criteria>
  <verify>`cd backend && rtk proxy python -c "from app.api.main import app, create_app; from app.main import app as compat_app; assert app is compat_app; assert callable(create_app); routes = {getattr(r, 'path', '') for r in app.routes}; assert '/health' in routes; assert '/ws/chat' in routes"`</verify>
  <done>Canonical app target exists and `app.main:app` remains compatible.</done>
</task>
