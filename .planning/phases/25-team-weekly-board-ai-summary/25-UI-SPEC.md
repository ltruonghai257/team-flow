> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, pending checker sign-off.

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (multiples of 4):

| Token | Tailwind Class | Value | Usage |
|-------|---------------|-------|-------|
| xs | `gap-1` / `p-1` | 4px | Inline icon gaps, compact badge padding |
| sm | `gap-2` / `p-2` | 8px | Meta rows, action clusters, chip spacing |
| md | `gap-4` / `p-4` | 16px | Default field spacing, card row separation |
| lg | `gap-6` / `p-6` | 24px | Section spacing between summary, composer, and board list |
| xl | `gap-8` | 32px | Desktop separation between major board zones |
| 2xl | `py-12` | 48px | Empty-state vertical rhythm |
| 3xl | `py-16` | 64px | Initial loading centering and sparse historical states |

Exceptions:
- Shared `.card` remains `p-5` (20px) because that is the established page/card padding in `/updates` and `/schedule`.
- Week navigation controls may use `px-3 py-2` for compact toolbar density.
- Icon-only controls must remain at least 36x36px hit area.

## Color

| Role | Tailwind Token | Hex | Usage |
|------|---------------|-----|-------|
| Dominant (60%) | `bg-gray-950` | #030712 | App background, page canvas |
| Secondary (30%) | `bg-gray-900`, `bg-gray-800`, `border-gray-800` | #111827, #1f2937 | Summary panel, post cards, editor shell, week navigation bar |
| Accent (10%) | `primary-500`, `primary-600` | #6366f1, #4f46e5 | Primary CTA, active week state, selected preview toggle, focus ring, summary refresh affordance |
| Destructive | `red-600` | #dc2626 | Delete post confirmation only |

Accent reserved for:
- `Summarize this week` primary CTA
- `Post Update` / `Save changes` primary actions
- Current-week chip or button state in the week navigator
- Focus rings on active inputs and textarea
- Summary cooldown/recently-updated status badges

Secondary semantic colors:
- Success / ready state: `text-emerald-300` with `bg-emerald-500/15` for “summary available” metadata only
- Quiet empty state: `text-gray-400` / `text-gray-500`
- Warning/cooldown: `text-amber-300` with `bg-amber-500/15` for cooldown notice only

Do not use accent color on every link or metadata chip. The weekly board should read like a digest, not a dashboard with competing highlights.

## Interaction Contract

### 1. Page Layout

`/board` uses a summary-first week digest, not a chat feed.

```text
/board
├── Page header
│   ├── Left: h1 "Weekly Board" + subtitle
│   └── Right: no floating CTA; compose lives in-page
├── Week navigator bar
│   ├── Previous week
│   ├── Current week shortcut
│   ├── Next week
│   └── Compact ISO week picker
├── WeeklySummaryPanel
│   ├── Selected week heading
│   ├── Stored summary or empty state
│   └── Summarize / regenerate action with cooldown feedback
├── WeeklyPostComposer
│   └── Create current-week post or edit own post
└── Weekly digest list
    ├── One primary card per member
    └── Optional follow-up append entries nested under the same member card
```

Page shell: `p-4 md:p-6 max-w-5xl mx-auto`.

Desktop: single dominant content column is preferred. A light two-zone split is allowed only for the summary/composer row if it remains visually consistent with existing TeamFlow pages.

### 2. Week Navigator

The selected week defaults to the current ISO week.

Structure:
- Left button: `Previous week`
- Middle button/chip: `Current week`
- Right button: `Next week`
- Trailing compact select or popover: historical ISO week jump

Visual rules:
- Container: `card` or `rounded-xl border border-gray-800 bg-gray-900 p-3`
- Active week state uses accent tint
- Non-active controls stay `btn-secondary`-like with gray emphasis

Behavior:
- Previous/next immediately reload the selected week payload.
- `Current week` always snaps back to this week.
- Historical picker labels use `2026 · W18` style and may append a short date range.
- Loading a week preserves scroll position at the top of the board page.

### 3. Weekly Summary Panel

This is the highest-priority surface on the page.

States:
- `loading`
- `empty` (no stored summary)
- `available`
- `cooldown-blocked`
- `no-updates`

Panel contract:
- Wrapper uses `.card`
- Title row: `AI Weekly Summary`
- Subhead shows selected week label and `updated_at` when available
- Summary body uses `text-sm leading-relaxed text-gray-300`

Behavior:
- If a stored summary exists, show it immediately for both current and past weeks.
- If no summary exists, show a quiet empty state with `Summarize this week`.
- If the week has no posts, show `No updates this week` and do not present AI-generated language.
- If cooldown is active, keep the stored summary visible, disable regeneration, and show precise remaining time.
- Regeneration overwrites the stored summary for that ISO week after success.

Action rules:
- Current week with no cooldown: primary button `Summarize this week`
- Current week with stored summary and no cooldown: secondary or ghost action `Refresh summary`
- Past weeks: allow viewing stored summaries immediately; regeneration may be hidden unless backend explicitly supports it for historical weeks

### 4. Weekly Post Composer

The composer exists on the same page and is scoped to the current week only.

If the member has not posted for the selected current week:
- Show a create panel titled `Write this week's update`

If the member already has a primary post for the selected current week:
- Replace create state with an edit/manage panel for their own post
- Keep follow-up append action separate from editing the primary post

If viewing a past week:
- Composer is read-only hidden
- Show a muted note: `Past weeks are read-only. Switch to the current week to post an update.`

Form fields:
- Markdown textarea
- Preview toggle with two states: `Write` and `Preview`
- Primary CTA

Preview rules:
- Toggle, not split-pane
- Preview uses `DOMPurify.sanitize(marked.parse(raw))` before `{@html}`
- Preview surface matches card styling and mobile width

### 5. Primary Weekly Post Card

Each member's primary post is rendered as one digest card.

Card order:
1. Author row: avatar fallback, name, timestamp
2. Primary markdown body
3. Author action row when owned by current user
4. Follow-up append entries
5. Add follow-up control when owned by current user and selected week is current

Visual rules:
- Wrapper: `card hover:border-gray-700 transition-colors`
- Markdown body uses `prose prose-invert max-w-none text-sm`
- Keep cards visually dense; avoid oversized blockquote/code styling unless present in content

Ordering:
- Posts should favor stable digest readability over noisy event order
- Default order: primary posts by first-created time ascending or author name ascending; append entries remain chronological under their parent post
- Do not interleave append entries between members

### 6. Follow-Up Append Entries

Append entries are part of the author's own weekly post model, not comments.

Rendering:
- Nested under the author's primary post with a subtle left border or inset section
- Each append entry shows timestamp plus markdown body
- Visual weight is lower than the primary post

Behavior:
- Author can add append entries only to their own current-week post
- Append entry composer is lightweight and collapsed by default
- Past weeks are read-only

### 7. Edit and Delete Contract

Author-only actions mirror `/updates`: no modal required by default.

Primary post edit:
- In-place card replacement with the same markdown textarea + preview toggle
- Buttons: `Discard changes`, `Save changes`

Delete confirmation:
- Inline confirmation row inside the card
- Copy must state that the weekly post and its follow-up entries will be removed for that week

Append entry edit/delete:
- Same inline treatment as the primary post, but visually confined to the append section

Another member's post or append entries:
- Never show edit/delete controls

### 8. Loading, Empty, and Error States

Initial page load:
- Existing centered spinner pattern from `/updates`

Week with zero posts:
- Heading: `No weekly updates yet`
- Body: `Be the first to post an update for this week.`

Past week with zero posts:
- Heading: `No updates were posted for this week`
- Body: `Choose another week or return to the current week to add an update.`

Summary empty state:
- Quiet card copy, not warning styling

Error handling:
- Toasts for mutation failures
- Inline retry state inside the summary panel for summary-generation failure

### 9. Mobile Contract

On mobile:
- Keep the page as one vertical digest
- Week navigator wraps cleanly without horizontal overflow
- Summary panel stays above composer and posts
- Preview toggle remains top-aligned inside the composer
- Action rows stack if needed; do not depend on hover

No live split markdown preview is allowed in this phase because it adds layout weight and reduces readability on smaller screens.

## Visual Hierarchy

### Page Priority Order
1. Selected week context
2. AI summary panel
3. Current-week composer or read-only notice
4. Member digest cards
5. Follow-up append sections

### Summary Panel Priority Order
1. Selected week label and summary title
2. Summary content
3. Summarize/regenerate CTA
4. Cooldown or last-updated metadata

### Density Guidance

This feature should feel like a compact weekly digest:
- restrained card radii
- quiet gray surfaces
- clear scan order
- accent color only where action state matters

Avoid:
- reverse-chronological chat styling
- oversized hero banners
- decorative gradients or illustration empty states
- multi-column masonry card layouts
