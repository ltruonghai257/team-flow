  <action>
Create `backend/app/routers/board.py` and register it in `backend/app/api/main.py`.

Implement a route set that covers:

- `GET /api/board/week`
  - defaults to the current ISO week
  - optional year/week query params for history
  - returns the grouped week payload

- `POST /api/board/posts`
  - current-week only
  - creates the caller's primary weekly post
  - rejects a second primary post for the same week

- `PATCH /api/board/posts/{post_id}`
  - author only
  - current-week only

- `DELETE /api/board/posts/{post_id}`
  - author only
  - current-week only
  - deletes child append entries with the primary post

- `POST /api/board/posts/{post_id}/appends`
  - author of the parent post only
  - current-week only

- `PATCH /api/board/appends/{append_id}`
  - author only
  - current-week only

- `DELETE /api/board/appends/{append_id}`
  - author only
  - current-week only

Use 403/404 behavior that does not leak another member's out-of-scope content. Do not add comment or reaction endpoints.
  </action>
  <acceptance_criteria>
    - `backend/app/routers/board.py` contains `prefix="/api/board"`
    - the router exposes a week payload endpoint
    - the router exposes primary-post CRUD endpoints
    - the router exposes append-entry create/update/delete endpoints
    - `backend/app/api/main.py` includes the board router
  </acceptance_criteria>
</task>
