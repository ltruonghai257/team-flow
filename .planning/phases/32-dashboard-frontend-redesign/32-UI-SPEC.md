| Property          | Value                                                         |

**Note:** Project uses global `.card`, `.badge`, `.btn-primary`, `.btn-secondary`, `.btn-danger`, `.input`, `.label` utility classes from `app.css`. All dashboard panels MUST use the `.card` class for visual consistency.

## Spacing Scale

Declared values (multiples of 4 — enforced via Tailwind):

| Token | Value | Tailwind Class | Usage                              |
| ----- | ----- | -------------- | ---------------------------------- |
| xs    | 4px   | `p-1`, `gap-1` | Icon gaps, inline padding          |
| sm    | 8px   | `p-2`, `gap-2` | Row padding, compact spacing       |
| md    | 16px  | `p-4`, `gap-4` | Default section padding, grid gaps |
| lg    | 24px  | `p-6`, `gap-6` | Panel padding, section spacing     |
| xl    | 32px  | `mb-8`         | Major layout gaps                  |
| 2xl   | 48px  | `py-12`        | Page-level breathing room          |
| 3xl   | 64px  | `py-16`        | Reserved / not used in this phase  |

Exceptions:

-   Panel header bottom margin: `mb-4` (16px) — matches existing `flex items-center gap-2 mb-4` header pattern
-   Avatar size: `w-8 h-8` (32px) for activity feed, `w-10 h-10` (40px) for team health cards — matches KpiScoreCard.svelte and StandupCard.svelte patterns

**Note (codebase pattern, not a spec declaration):** Existing task row uses `p-2.5` in the current `+page.svelte`. The executor should match this existing pattern for the My Tasks rows to maintain visual consistency with the rest of the app. This is carried from existing code, not newly declared here.

## Typography

| Role            | Size | Weight | Line Height | Tailwind                            |
| --------------- | ---- | ------ | ----------- | ----------------------------------- |
| Body            | 14px | 400    | 1.5         | `text-sm`                           |
| Label / Subtext | 12px | 400    | 1.4         | `text-xs`                           |
| Panel Heading   | 16px | 600    | 1.3         | `font-semibold text-white`          |
| Page Title      | 24px | 600    | 1.2         | `text-2xl font-semibold text-white` |

**Declared weights: 2 — Regular (400) + Semibold (600)**

Exception (pre-existing, not introduced in this phase): page h1 carries `font-bold` (700) from current dashboard — executor must not introduce additional bold weights.

**Type scale in use for this phase:**

-   Page title: `text-2xl font-bold text-white` — "Dashboard" h1 (unchanged)
-   Panel section headers: `font-semibold text-white` — "My Tasks", "Team Health", "Activity Feed", "KPI Summary"
-   Task title: `text-sm text-gray-200 truncate`
-   Project name subtext: `text-xs text-gray-500`
-   Badge text: `text-xs font-medium` (inside `.badge` class)
-   KPI metric number: `text-2xl font-semibold` color-coded per threshold
-   Empty state copy: `text-gray-500 text-sm py-4 text-center`

## Layout Structure (from D-01, D-02, D-03)

### Focal Points

-   **Member view focal point:** My Tasks panel — primary action surface, first element rendered, draws eye with urgency signals (overdue tint, due-soon tint)
-   **Supervisor view focal point:** KPI Summary Strip — top-of-page, 3-card row with color-coded avg score number acts as primary visual anchor for at-a-glance health check

### Supervisor / Manager / Assistant Manager view:

```
[KPI Summary Strip — 3 cards, full-width]
[My Tasks (left) | Activity Feed (right)] ← lg:grid-cols-2
[Team Health Panel — full-width]
```

### Member view:

```
[My Tasks (left) | Activity Feed (right)] ← lg:grid-cols-2
```

Role gate: `{#if $isManagerOrLeader}` — wraps KPI strip and Team Health panel.

### Mobile (< `md` breakpoint):

All sections stack single-column, same order as desktop. KPI strip collapses to 1-col. Team health grid uses 2-col (`grid-cols-2`).

## Accessibility Contract

| Concern            | Implementation                                                                               |
| ------------------ | -------------------------------------------------------------------------------------------- |
| Color-only signals | Status dots supplement with text count ("2 active · 1 overdue")                              |
| Link text          | Descriptive: "View full performance", "View all updates", "View all tasks" — not bare arrows |
| Heading hierarchy  | h1 page title → h2 panel headings (no h3 in this phase)                                      |
| Badge ARIA         | `.badge` elements — no additional ARIA needed (decorative)                                   |
| Keyboard           | All interactive rows are `<a>` or `<button>` — keyboard accessible by default                |
